Art+Code

Interactive Programming

I have been experimenting with different techniques in search for my own. I have always been attracted to new technologies and seeing virtual worlds emerge from multiple lines of code.

I started developing interactive experiences and artwork with Microsoft Kinect at the end of 2011 with the use of OpenFrameworks, OpenGL and various hardware inputs. Since then, I have experimented with engines like Unity3D and more platforms like Oculus Rift, Leap Motion, PS Moves and projection mapping.

One of my simulations was made for an interesting project called S(aroud) with the architecture department. S(around) was an amazing experience. It exposed me to the potential and the amount of possibilities architecture installations have. Working with architects was great because it challenged my technical skills by making sure new experimental ideas where possible and finished in a tight schedule. I liked it so much that I am currently doing another project with the architecture department. This time it consists of a dancing performance with the use of projection mapping and the use of Arduino and kinect to detect motion.

  • (Nov. 2012)RPI

    Sensor Path:

    An arduino project combined with openFrameworks. I used a flex sensor and a soft potentiometer to create a path in which each time the direction was changed, a circle would represent the place where the direction changed. To finish up, I added an RGB LED so the user could see the color the circle was going to be.
    The flex sensor is an input that determines how much you are flexing it. It was used to change directions. Depending how much it was bend, the angle in which the direction would change would be determine. It also determined the radius of the circles. The greater the angle the flex would calculate, the bigger the radius of the circle. The soft potentiometer is a neat input device that detects pressure along its length. When you press it down with a finger it will change resistance depending on where you're pressing it. In this project I used it to determine the color of the circle. The bar was divided in three regions; red, green and blue.

  • (Oct. 2012)RPI

    Point Cloud Me:

    Point Cloud Me is a project I did inspired by the video of “point cloud dancer” . After seen the video, I wanted to do something similar but in real time. I decided to make Point Cloud Me sound reactive to give the user some control on what would happen throughout the simulation. If you do a low pitch sound, you decompose as sand and if you do a high pitch sound, you come back together. In the mean time you have particles falling from your arms to represent the destruction of the “point cloud dancer” video. Enjoy! :)
